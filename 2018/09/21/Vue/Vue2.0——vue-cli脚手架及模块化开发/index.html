vue-cli脚手架

vue-cli就是快速搭建环境的，它有几个模板，webpack的，browserify的，pwa的等等。都是使用npm来操作的。通常使用较多的时webpack吧。

  1. 安装

cnpm安装

npm install -g cnpm --registry=https://registry.npm.taobao.org

vue-cli安装

cnpm install vue-cli -g     //-g为全局安装

  1. 查看命令

    vue //vue命令
    vue -V //版本
    vue list //可用的模板

  1. 初始化项目

    //初始化 模板名 项目名
    vue init webpack vue-cli-demo

首先这个流程，如果你项目名称不符合规范，比如长度，那么么第一条就让你重新定义项目名，然后是项目描述，作者，然后是运行时编译，vue-router不安装，ESLint是代码格式规范，unit test为单元测试，e2e test也是测试，最后一个npm install 选择no最后一项。

  1. 安装模块

就是流程完后，后面提示的黄色代码,但是我们不使用npm，而使用cnpm

cd vue-demo-test
cnpm install    //安装依赖，虽然已经初始化项目了，但是依赖是没有安装的。

  1. 运行示例

    npm run dev //启用测试服务，开发环境下
    npm rum build //构建项目，生产环境，将项目打包输出到dist目录下。

模块化开发

vue-router

  1. vue-router模块化开发

    cnpm install vue-router -S //–save,写入dependencies，用于生产环境

  1. 编辑main.js 导入vue-router组件，并使用

    import VueRouter from ‘vue-router’

    Vue.use(VueRouter)

  1. 创建components文件夹，下面放所有的组件

创建router.config.js文件，用于配置路由js，专门配置路由的。引入组件并进行配置

  1. main.js中创建router示例,并在Vue示例中设置

路由守卫 监视路由

watch:{
    $route:function(newVal,oldVal){

    }
}

newVal和oldVal分表表示路由的新状态和前一个状态。

axios模块

cnpm install axios -S

和vue-router不一样，axios不能使用Vue.use()来引入形成全局的 所以有两种方式的引入:

  • 每个组件都引入这个组件

  • 在main.js中引入，并将组件添加到Vue的原型上

    Vue.prototype.axios = axios;

为自定义组件添加事件

当你在根组件定义事件，根组件模板中的按钮可以触发事件。自定义组件时，无法绑定其事件，需要.native修饰符

Element UI的引入，由饿了么开发的Vue UI框架

  1. 安装

    cnpm install element-ui -S

  1. 引入

    //main.sj中引入
    import ElementUI from ‘element-ui’
    import ‘element-ui/lib/theme-chalk/index.css’

注意，elementui 2.0的版本index.css位置已经变了

  1. 配置style-loader

    //安装前首先确认style-loader和css-loader是否下载
    cnpm install stype-loader -D
    cnpm install stype-loader -D

  1. 安装less

安装loader,less、less-loader

cnpm install less less-loader -D

build/webpack.base.conf.js下rules添加

{
    test: /\.less$/,
    loader: 'less-loader'
}
