部署到github

  1. 在项目根目录下_config.yml配置文件配置参数。在文件最后面添加参数。此文件为yml文件，注意缩进格式

  1. 安装一个部署插件hexo-deployer-git

    此条命令并非全局安装，需要在hexo项目根目录下执行，注意了。

  2. 部署上去

    g表示generate生成,-d 参数表示deploy部署。

    然后访问网址: https://用户名.github.io

写作文章

执行完毕后，会提示文件在哪，在/source/_posts下有一个这个文章标题的md文件。

你可以编辑md文件，Markdown语法就不多说了。每次创建完之后，在文件最上面都有文件元信息,由三个-符号上下包裹

默认是这个，但是还可以添加更多的内容。这个元信息表示这个文件的所属信息，在主题中会使用到的。包括tags属性名和categories属性名，分别表示标签和分类。

Ctrl+C可以退出hexo服务。执行第一个命令可以重新生成相关文件，执行第二个命令启动服务。

如果要再次部署到github上，需要清一下缓存或历史文件

另外，如果你的文章暂时不发布可以先保存在草稿里面。生成草稿的方法和文章差不多 hexo new draft “文章标题”，生成后会在 /source/_drafts 里看到你的草稿文章。当你想发布时只要执行 publish 命令即可发布到博客。

主题使用

我们以当前站点主题为例，访问hexo主题,找到A-Ayer,这个是实例页面,我们可以看一下相关说明,如何使用。

  1. 下载主题，直接进入项目根目录就行，命令已经指定了下载位置，不需要进入到themes里。

  2. 修改根目录下的_config.yml文件,修改主题值为alyer

    然后，你可以重新刷新一下页面或clean，重新访问即可看到新主题

  3. 目录的创建Categories

    此时_source/categories/index.md就会出现。然后修改index.md的源信息,这就行了。不要往里面添加任何内容。

  4. 创建tags标签

    此时_source/tags/index.md就会出现。然后修改index.md的源信息,这就行了。不要往里面添加任何内容。

  5. 完毕后，创建文章，只需要修改文章的原信息即可,在元信息上添加这些属性,标签是数组形式，分类只能所属一个

    这样，再看分类和标签就有东西显示出来了。

如果你打开页面的话，如果没有任何一个文章有标签5的配置，那么标签和分类页面都是空的。另外，你需要注意一下主题里的_config.xml，可以配置一下menu菜单和其他相关的信息。更多操作可以在说明文档或作者dev分支项目中看到目录分级和操作。
