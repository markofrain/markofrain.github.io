数据查询语言

SELECT数据查询，可以检索数据，由一系列字句组成，根据字句条件满足情况查询，按照复杂程度可分为简单查询、WHERE条件查询、多表查询、子查询等。

注意：ORACLE中语句字母大小写无影响，字段均转换为大写。建议关键字大写。

语法结构:

SELECT 
    [DISTINCT|ALL]
        select _list
    FROM table_list
    [where_clause]
    [group_by_clause]
    [HAVING condition]
    [order_by_clause]

语法说明：

  • SELECT：查询关键字
  • [DISTINCT|ALL]：字段数据中的数据是否去除重复记录
  • select_list：需要查询的字段列表
  • FROM：查询必须关键字，数据来源
  • [where_clause]：查询的WHERE条件部分
  • [group_by_clause]：GROUP BY 子语句部分
  • [HAVING condition]：HAVING字句部分,排序后筛选

简单查询

1.指定字段查询

SELECT productId,productName,productPrice FROM product;

各字段使用逗号分割。 2.获得所有字段数据

SELECT *  FROM product;

使用星号省略所有字段，但这样做导致查询缓慢，在生产环境中并不建议使用。 3.使用别名代替字段

SELECT productId as 产品编号,productName as 产品名称,productPrice as 产品价格 FROM product;

别名一般使用在开发环境中，生产环境也不适用。 4.使用表达式操作字段

SELECT productId,productName,productPrice || '*' || 1.25 || '=' || productPrice*1.25 as new_productPrice FROM PRODUCT;

语句中有“||”表示字符连接符号和“”符号表示算数乘号 *5.使用函数操作查询字段**

SELECT count(*) FROM product;
或
SELECT count(0) FROM product;

SELECT lengthb(productName) FROM product;

count函数查询表中记录，lengthb获得字符串字节数。 6.去处检索数据中重复记录

select distinct productName,productPrice from product;

如果我再插入一条除主键外相同记录，那么此语句将根据distinct后的字段列进行去重,若查询出来有相同的，则以第一条为准。如果字段列有主键，那么永远不会重复。

WHERE条件查询

WHERE字句用于条件筛选，对字段进行筛选,满足条件查询。 WHERE字句中可以使用的操作符有关系操作符、比较操作符、罗技操作符。 1) 关系操作符包括：<、、>=、=、!= 2) 比较操作符包括:

  • IS NULL：操作数为NULL返回TRUE。
  • LIKE：模糊匹配比较字符串。
  • BETWEEN..AND…：在两值范围中。
  • IN：验证操作数在一系列值中。

3) 逻辑操作符:

  • AND：并且，两个条件都满足
  • OR：或者
  • NOT：某个逻辑值取反

1.单一条件查询

SELECT * FROM product WHERE productPrice>10;
SELECT * FROM product WHERE productName != '晨光签字笔'

通过比较运算符，比较,字符要用引号引起来。 2.多条件查询

SELECT * FROM product WHERE productId>2 AND productWeight IS NULL;

SELECT * FROM product WHERE productId>2 OR productWeight>10;

SELECT * FROM product WHERE NOT productWeight<10;

SELECT * FROM product WHERE NOT productWeight IS NULL;

SELECT * FROM product WHERE productPrice BETWEEN 10 AND 50;

3.LIKE模糊匹配查询

  • _：可以代替一个字符

  • %：可以代替多个字符

    SELECT * FROM product WHERE productName LIKE ‘晨光%’;

    SELECT * FROM product WHERE productName LIKE ‘_光%笔’;

4.限制条件在某范围内

SELECT *  FROM product WHERE productId IN (1,3,4);

它将筛选出productId为1,3,4的记录,一般IN用于子查询中，而且括号中的值一般都为主键值。

分组查询

聚合函数

  • AVG(column)：列所有值的平均数

  • SUM(column)：列所有值的总和

  • MAX(column)：列所有值中的最大值

  • MIN(column)：列所有值的中的最小值

    SELECT AVG(productPrice),SUM(productPrice),MAX(productPrice),MIN(productPrice) FROM product;

GROUP BY子句用于分组，HAVING子句用于分组后的筛选。 笔记:

  • 聚合函数只能出现在查询列表、HAVING、ORDER BY 子句中(不能出现在WHERE子句中)。

  • 如果再SELECT语句中关键字排序为GROUP BY,HAVING、ORDER BY、LIMIT

  • 在查询列表中如果有列、表达式、聚合函数，那么这些列和表达式必须出现在GROUP BY子句中，否则出错

    SELECT SUM(productPrice),productName FROM product GROUP BY productName;

按产品名称分组，获得该分组每个Name值对应价格的和。如果查询列表出现productName则GROUP BY中必须有productName，如果查询列表没有，那么GROUP BY子句中也可以有。 当然GROUP BY子句中可以有多个列名。 HAVING子句放在GROUP BY后面，用于WHERE之后分组完再次筛选，与WHERE用法一致

SELECT SUM(productPrice),productName FROM product GROUP BY productName HAVING SUM(productPrice)>30;

检索结果排序

ORDER BY子句用于排序操作,字段名后跟排序方式，ASC表示升序，DESC表示降序

SELECT *  FROM product ORDER BY productPrice;

SELECT SUM(productPrice),productName FROM product GROUP BY productName ORDER BY SUM(productPrice) DESC;

数据定义语言数据库表的创建

前面已经有简单的了解数据定义语言，那么直接切入正题。 在创建表之前，先说一下。数据库的问题。 与MySQL数据库不一样，mysql可以创建多个数据库，而Oracle只有一个数据库orcl，它通过用户来操作，每个用户访问到的数据是不一样的。你需要创建用户，创建并分配表空间，赋予权限。并登陆创建的用户来操作。在后面讲，他与sql操作无关，可跳过学习。

创建表

创建语法:

CREATE TABLE table_name
(
column_name datatype [null|not null],
column_name datatype [null|not null],
...
[constraint]
)

语法说明:

  • table_name：数据表名称，在同一数据库中数据表名是不能重复的
  • column_name:列名，或字段名，在一个表中不能重复
  • datatype:数据类型，上节课说的
  • [null | not null]:表明该列是否为空或者不为空
  • [constraint]:设置表中列的约束，主要包括主键约束、外键约束、检查约束等。

示例:

CREATE TABLE product
(
productId number(4) not null,
productName nvarchar2(20),
productPrice number(10,2),
CONSTRAINT pk_product primary key(productId)
)

语法已经说了，约束上章已经讲了，代码就不解释了。

删除表

示例：

DROP TABLE table_name;

删除整张表，删除后将无法使用。

ALTER 语句修改表

如果想更改表的属性或是列，可使用LATER TABLE语句 语法:

ALTER TABLE table_name 
ADD column_name | MODIFY column_name | DROP COLUMN column_name

语法说明:

  • ADD:用于向表中添加列
  • MODIFY:用来修改表中已存在的信息
  • DOP COLUMN:删除表中的列删除列时有时要加上CASCADE CONSTRAINTS,把该列有关的约束一并删掉。

示例:

//假如还是上面创建的product表,要为其添加productWeight重量字段
ALTER TABLE product ADD productWeight number(6,2);

//修改productName的数据类型为varchar2(40),查看数据类型是否修改看下一个代码块
ALTER TABLE product MODIFY productName varchar2(40);

//删除列productWeight字段
ALTER TABLE product DROP COLUMN productWeight;

//更改列名,productName改为productN
ALTER TABLE product RENAME COLUMN productName to productN;

通常删除列字段可能会到你没有权限,无法删除属于sys的表中的列

//查看表数据结构
SELECT table_name, column_name, data_type, char_length
  FROM all_tab_cols
 WHERE table_name = 'PRODUCT' ;

这是表的特性的增删改字段操作，而通过ALTER TABLE添加约束，上个文章中也是说了的。

数据操作语言

INSERT语句

INSERT语句是对表数据的新增操作. 语法:

INSERT INTO table_name(column1，column2，column3...) VALUES(val1,val2,val3...);

如果你要插入整个列的字段,那么不必写列名了

INSERT INTO table_name VALUES(val1,val2,val3...);

当然如果主键是自增的,那就不需要添加主键列了，但是需要为主键列设置序列(后面讲)。 示例:

INSERT INTO product(PRODUCTID,PRODUCTNAME,PRODUCTPRICE,PRODUCTWEIGHT) VALUES(1,'晨光笔',12,20);
INSERT INTO product VALUES(2,'怡宝矿泉水',4,1500);

DELETE语句

DELETE语句用于表数据的删除，根据条件进行删除，一般通过获得主键值删除。 语法:

DELETE FROM table_name WHERE condition

示例:

DELETE FROM product WHERE productId=1;
DELETE FROM product WHERE productName='怡宝矿泉水';

WHERE条件后可以添加任何条件。 如果你想删除整个表格数据，而不删除表结构，可使用如下语句

TRUNCATE TABLE table_name;

区别:

  • DELETE：删除表数据会记录日志，相对比较慢,commit前是可以回滚恢复的
  • TRUNCATE：不记录日志，永久删除，数据不能恢复，速度快，但不会删除表结构
  • DROP TABLE：直接删除整个表，不可恢复

UPDATE 语句

进行数据更改的操作 语法:

UPDATE  table_name SET column1=val1[,column2=val2....] WHERE [condition]

示例:

UPDATE product SET productWeight=550,productName='康师傅小瓶矿泉水' WHERE productId= 2;

一般更改条件根据主键获得，而一般不会更改主键，因为这会导致不好的情况。

约束的使用

主键约束

主键在每个表中只有一个，主键列是唯一标识，它的值是唯一的不可重复，不能为空。 例如像产品表中，productId,productName，productPrice,productAddress等，productId就相当于主键，用来唯一标识这条数据。 1.可在创建表语句中通过primary key(字段名)设置 2.还可以使用ALTER TABLE语句为表添加主键约束和移除主键约束。

//创建时添加
CONSTRAINT constraint_name PRIMARY KEY(column_name)
//添加约束
ALTER TABLE table_name ADD CONSTRAINT constraint_name PRIMARY KEY(column_name);
//移除约束
ALTER ATBLE table_name DROP CONSTRAINT constraint_name;
  • constraint_name:约束的名称
  • column_name:作为主键列的列名

外键约束

1.外键是指在表（子表）中定义外键的字段的列的值必须在另一张表（父表）中出现。外键是用来实现参照完整性的方法之一。 2.被参照的表称为父表，创建外键的表称为子表。子表中的外键关联的是父表中的主键 3.外键可以在建表的时候定义，也可以通过alter table语句来创建。 例如:班级和学生,为学生添加约束前需已经创建了主表Class表,否则无从参考，在学生表中添加约束，外键列为学生表里的class_id,参考的表为Class，参考的列为Class的id。当class的id删除之后，学生表中关联该id的记录都会被级联删除。

//在创建表中添加
CONSTRAINT constraint_name FOREIGN KEY (column_name) 
REFERENCES table_name (column_name)
[ON DELETE CASCADE];
//创建表之后添加
ALTER TABLE table_name ADD CONSTRAINT FOREIGN KEY (column_name) REFERENCES table_name (column_name);
//删除约束
ALTER TABLE table_name DROP CONSTRAINT constraint_name;

  • constraint_name:创建的外检约束的名称

  • FOREIGN KEY(column_name):指定外键约束的列名(子表外键列名)

  • table_name(column_name):要引用的表名,列名(主表主键名)

  • [ON DELETE CASCADE]:可选，设置级联删除，当主键的字段被删除，外键对应的记录也被删除。

    CONSTRAINT FK_class_student FOREIGN KEY(classId) REFERENCES class(Id) ON DELETE CASCADE;

    ALTER TABLE student ADD CONSTRAINT fk_stu_class FOREIGN KEY (classId) references classes (Id) ON DELETE CASCADE;

    ALTER TABLE student DROP CONSTRAINT fk_stu_class;

检查约束

检查约束就是CHECK约束，规定该列能输入的值，以保证数据的正确性 1.在创建表时添加约束 2.使用ALTER TABLE添加约束

CONSTRAINT constraint_name CHECK(condition);

ALTER TABLE table_name ADD CONSTRAINT constraint_name CHECK(condition);

ALTER TABLE table_name DROP CONSTRAINT constraint_name;

condition指约束条件,可以使用between and,><,and,or等。

CONSTRAINT chk_age CHECK(age>=18 and age<=60);
ALTER TABLE employee CONSTRAINT chk_sex CHECK(sex='男' or sex='女');
ALTER TABLE employee CONSTRAINT chk_age CHECK(age between 18 and 60);

唯一约束

UNIQUE约束也称为唯一约束，设置后表示该列的值时唯一的，不允许重复，可以为空格，一个表中可以有多个唯一约束的列 1.在创建表时添加约束 2.使用ALTER TABLE添加约束

CONSTRAINT constraint_name UNIQUE(column_name);
ALTER TABLE ADD ....//与上面的格式一致
ALTER TABLE DROP

非空约束

NOT NULL约束，表示该列的值不能为空 1.在创建表时，在列名后面写 2.通过ALTER TABLE添加删除约束

//创建时添加,不写后面的NOT NULL则默认可以为空
empName nvarchar2(10) NOT NULL
//创建后设置为NOT NULL
ALTER TABLE table_name MODIFY column_name NOT NULL;
//创建后设置为NULL
ALTER TABLE  dept MODIFY name NULL;

CDN简介

内容分发网络(Content Delivery Network，简称CDN),是建立并覆盖在承载网之上、由分布在不同区域的边缘节点服务器群共同组成的分布式网络，替代以Web Server为中心的传统数据传输模式。 CDN的工作原理就是将您源站的资源缓存到各地的边缘节点服务器（CDN节点）上，用户请求访问和获取资源时，就近调用CDN节点上缓存的资源。这种分布式数据传输方式，使得用户请求的资源不需要都回您的源站获取，从而避免网络拥塞、分担源站压力，保证用户访问资源的速度和体验。 不多说功能其他文字叙述了。 简而言之它能加速网站内容的加载速度，从而缩短网页加载时间。通俗的讲就是，变快。 使用场景: 使用场景和相应的配置，是根据你自身站点的业务来选择的。 阿里云CDN主要提供了以下五种业务类型。

  • 图片小文件加速(网站应用静态加速)
  • 大文件下载
  • 音视频点播加速
  • 直播流媒体加速
  • 全站加速
  • 移动加速

对于目前的小站点，博客，论坛等。不应用视频，下载方面的站点，可以选择图片小文件加速。 站点中有动态资源(php,aps等)和静态资源(img,js,css等)，对于大量静态资源的加载，需要静态分离。动态资源放在服务器上，静态资源使用阿里云CDN加速，可以结合OSS对象存储使用。 可以有效地搞定网站图片，短视频等内容分发。 阿里云CDN帮助文档的架构图: 接下来就开始动手做。

开通CDN

在开启阿里云CDN服务之前，确保你的域名是一定要备案通过的。 1.开通CDN服务 首先进入阿里云的CDN产品详情页,你可以查看产品价格，以及帮助文档。计费分为两种，按带宽峰值和按流量。根据在产品价格页面查看，按理来说，按流量应该便宜一点吧。你也可以选择CDN流量包。 在你进入产品详情页后，有立即开通，这就是开通服务入口，选择按流量开通，如果服务已经开通，则之即进入CDN控制台。服务开通后你就可以选择左边的流量包套餐，根据情况选择。 2. 添加加速域名 进入控制台后，选择左侧的域名管理或者右边的添加域名。填写你要加速的域名,业务类型就是图片小文件，源站类型选择IP,并输入你的服务器IP地址，端口默认80，点击下一步。域名将添加完成。 可能需要一到两天的审核，在域名管理的列表中，状态栏会显示审核中。不过很快，十分钟左右不到就显示正在运行了。不过此时还没有加速，审核完毕后，会给你一个CNAME值 点击配置按钮,确保你的回源设置中，源站设置的值是你的服务器地址:80,回源host的值设置成你的加速域名。 3. 配置域名解析CNAME 在解析之前，你需要知道，A解析与CNAME解析会存在冲突，在此处你可以了解解析规则冲突 当你确保你的第2步没有问题后，打开云解析CDN控制台，找到你的加速域名对应解析。不要加速域名是fsats.com,解析到xiaoxx6.com里了。点击添加解析。 主机记录:

  • 如果你在加速域名填写xxx.com那么此处写@
  • 如果加速域名填写.xxx.com,那么此处写
  • 如果加速域名写www.xxx.com，那么此处写www

记录值填写，CDN控制台加速域名审核通过后那个CNAME值，然后点击确定。 如果提示你，与A类型解析冲突，那么你就删除A记录类型吧。 4. 查看是否生效 不同服务商，响应时间不同。通过cmd，ping 加速域名,如果被转向 .kunlun*.com,则解析生效。 等待几分钟，等等那边服务处理完毕，你就可以访问了。 具体阿里云的帮助文档你可以访问你这里还有这里

购买OSS对象存储并创建存储

1.在阿里云选择产品，对象存储OSS,购买资源包。 选择存储包，全国大陆通用，选择你的存储包规格，选择你的时长。根据你的需求进行购买。 2.回到控制台，点击创建Bucket,输入名称，查看旁边的命名规范，设置区域，选择你的存储类型，下面都有解释的。 创建完毕。 3.设置你的bucket的读写权限，默认创建后是私有。点击基础设置，读写权限，设置，选择公共读，这样在显示图片时可以通过宽高调整，但没有写的权限。

添加子账户

现在访问的话，你使用的是主账户，其Access Key和Secret Key具有操作阿里云账户的所有权限，所以为了安全你要根据你的需要禁用这个账户并创建一个新的子账户，并赋予子账户操作OSS的权限。 1.触碰你的头像，点击Accesskey，进入控制台。它会自动姿势你创建一个新的子账户，你可以选不创建，那么么进入控制台后将显示你的主账户，你可以禁用它。 2.重新进入控制台，点击弹出来的，创建子账户，然后点击新建用户，完善表单信息，点击复选框生成Accesskey。 3.创建完成后，保存你的AK信息，将会以vcs格式的方式保存到你的本地，可通过excel表格打开,一定好好保存，不要泄露。 4.刚创建出来的用户，点击右边的管理，点击左侧的用户授权策略，点击右上角的编辑授权策略,选择OSS对象存储，添加到你的权限中，然后确定，设置完毕。 5.如果你要删除用户，则返回到用户管理，点击对应用户后面的删除按钮,选择复选框删除用户并撤销权限。

访问防盗链设置

你可以设置防盗链,只有该网站路径下发出的请求才能访问OSS存储，具体看旁边文档。 你可以设置成你的网站路径，这样只有在你的网站上发出请求才能访问。 该OSS对象存储，都是以文字的方式叙述，但步骤则是很完全的，如果你看不太懂，则在下方留言，我会随时关注并给予反馈。

后台配置

下载完毕后，通过后台的插件系统，上传插件，并启动。然后会在settings中出现 OSS Upload选项，点进去就可以设置。 1.勾选启用OSS最为媒体库 2.设置你的Access Key和Secret Key 3.上传路径设置为框处的格式，oss://存储名称,还可以在后面加上一个目录{path}。但是呢你的原始上传本地库，settings设置里设置了文件上传位置和目录。所以他会根据每年和每月作为目录存储，所以最终的文件在，访问域名/{path}/year/month/文件。 4.访问链接就是你的存储区域的访问域名,可以在OSS管理控制台中看到 5.上传端域就是endpoint，就是你的访问域名去掉bucketName，控制台上也有。如果你的服务器所在区域和OSS所在区域一致，就是机房位置，比如，都在华北2(北京)或者华北1(青岛),那么么可以内网访问，这样可以免去一些流量，可以设置内网的域名。具体的情况你可以看 使用文档或者表单右边的小问号。 1.图片缩略图设置，你可以选择第三个，介绍一下这几个:

  • 第一个就是原图，你上传什么大小，就显示什么大小，如果图片较大，则加载页面时耗费的时间的流量也比较大。
  • 第二个则生成缩略图，不光上传原图，而且还生成缩略图，占用更多空间，且扰乱文件结构
  • 第三种则是，上传原图，通过图片服务，你可以设置参数，规定图片显示的大小，且不生成缩略图，花括号中的width和height就可以更改显示图片的宽高
  • 第四种是，你可以为显示的图片设置一个样式，这样访问图片是携带样式，样式设置可以为图片添加水印啊，等各种图片的设置，你可以访问OSS管理控制台，图片处理选项。然后查看文档。

2.然后后面有备份的选项，即，上传到OSS，又在本地服务器上传一份。如果你的空间大小不大，你可以不选择。 3.配置这几个就可以了，其他的就不多说了，自行测试。
