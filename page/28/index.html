    比如以jsp为例，初始进入页面不仅需要json请求配置option，又要对象属性赋值，因此在select中的option添加完毕之后$("select[name='unit']").html(optionhtml)，再执行回显的js才可以回显成功。

  4. 包括搜索配置，清除按钮，选择完不自动关闭，排序等，以及直接ajax属性请求数据显示。

帆软相关问题

基于8.0低版本问题

  1. 为文本控件设置默认值，控件值为空设置默认值。当点击控件后，设置的默认消失了。原因是设置了控件的值长度。取消长度规则设置，若需要长度校验可在控件属性-编辑后时间中处理。

  2. 当你编辑控件文本框时，如果没有点击空白区域，那么控件不会认为你已经编辑结束，且输入的值，会在编辑结束后才会被控件接收到。因此当你编辑完控件值后，直接点击保存，那么得到的值是上一个编辑完成后的值。

  3. 报表显示高度问题。当最后一行没有单元格被占用是，报表就到结束了。如果你想获取某单元格的数据放到(动态js)另一个地方，以便其他操作的时候可以取，那么这个时候就不能把值放到报表无数据最后面。因为现实报表后，没有行占用。就没有这一样，那么你要获取的话就获取不到。

常用命令

拉取镜像

  • docker search [image_name]
  • docker search -s 30 [image_name]:点赞数stark超过30的当前镜像名称
  • docker search [image_name] –automated:查询有自动构建的镜像
  • docker pull [image_name][:TAG]:拉取镜像，或添加镜像标签版本

镜像操作:

  • docker images -a:列出本地所有镜像(镜像是分层的)
  • docker images -qa:列出所有本地镜像的镜像id
  • docker images –digests：查询所有镜像的详情
  • docker images –no-trunc:查询所有镜像的完整信息
  • docker rmi image_name[:TAG]:删除镜像
  • docker rm container_id:删除已停止的容器
  • 批量删除容器:docker rm -f ${docker ps -a -q}

运行命令

  • –name=”容器名字”:为容器指定名字
  • -d:后台运行容器，并返回容器ID，守护式进程
  • -i:以交互模式运行容器，通常与-t同时使用
  • -t:为容器分配一个伪输入终端，通常与-i一起使用
  • -P:随机端口映射
  • -p:指定端口映射，格式如下:
    • ip:hostPort:containerPort
    • ip::containerPort
    • hostPort:containerPort
    • containerPort

ps命令

  • docker ps -a:列出当前正在运行的和历史上运行的容器
  • docker ps -l:显示最近创建的容器
  • docker ps -n [num]:显示最近启动的n个容器
  • docker ps -q:静默模式，只显示容器编号
  • docker ps –notrunc:不截断输出

退出启动容器

  • exit:容器停止退出
  • ctrl+Q+P:容器不停止退出
  • docker start container_id:启动容器
  • docker restart container_id:重启容器
  • docker stop container_id:停止容器
  • docker kill container_id:强制关闭容器

其他操作

  • docker logs -f -t –tail container_id: 日志查看

    • -t:加入时间戳
    • -f:跟随最新的日志打印
    • –tail 数字:显示最后多少条

  • docker inspect container_id:查看容器内部运行细节

  • docker exec -it container_id ls -l /tmp:不打开终端，直接打印出容器内部命令

  • docker exec -it container_id /bin/bash:打开新终端，启用一个进程

  • docker attach container_id:直接进入启动命令的终端，不会启动新的进程

  • 从容器内文件拷贝到本地:docker cp container_id:/path localpath

Docker镜像操作

镜像原理

镜像是轻量级、可执行的独立软件包，用来打包软件运行环境和基于环境开发的软件，它包含某个软件所需的所有内容，包括代码，运行时，库，环境变量和配置文件。

联合文件系统(UnionFS),是一种分层、轻量级并且高性能的文件系统，它支持对文件系统修改作为一次提交来以层层叠加，同时将不同目录挂载到同一个虚拟文件系统下。UnionFS是Docker镜像的基础。镜像可以分层进行继承，基于基础镜像(没有父镜像)，可以制作各种具体的应用镜像。

bootfs主要包含bootloader和kernal，bootloader主要是引导加载kernal，linux刚启动时会加载bootfs文件系统，在Docker镜像最底层是bootfs。这一层与我们经典的Linux/Unix系统是一样的，包含boot加载器和内核。当boot加载完成之后整个内核都在内存中了，此时内存的使用权由bootfs交给内核，此时系统卸载bootfs。

rootfs，在bootfs之上，包含经典linux系统中的/dev,/proc,/bin等目录文件。

对于一个精简的OS，rootfs可以很小，只需要包括基本的命令，工具和程序库就可以了，因为底层直接使用Host和kernal，自己只需要提供rootfs就可以了。因此不同的发行版本可以共用bootfs

资源共享：多个镜像由多个base镜像构建而来，那么宿主机秩序保存一份base镜像，可以被共享。就是是说，多某个镜像的基础镜像在之前的镜像下载中已经下载过，那么就不需要下载。整个镜像是一层层叠起来的。就像下载是，下载的是多个镜像，而不是tomcat这一个单一的镜像。

保留字指令

保留字指令必须大写

  • FROM：将要编辑的基础镜像
  • MAINTAINER:镜像维护者的姓名和邮箱
  • RUN:容器构建时需要运行的命令
  • EXPOSE：对外暴露的端口
  • WORKDIR：创建容器后，默认进入的工作目录
  • ENV:构建镜像时，设置环境变量
  • ADD：将宿主机下的文件拷贝到镜像且ADD命令会自动处理url和解压包
  • COPY：拷贝文件和目录到镜像
  • VOLUMES: 容器数据卷，用于数据保存和持久化工作
  • CMD:指定容器启动时要运行的命令。dockerfile中可以有多个CMD命令，但只有最后一个生效，CMD会被docker run之后的参数替换
  • ENTRYPOINT：指定一个容器启动要运行的命令。当添加参数时，会进行参数合并
  • ONBUILD：当构建一个被继承的dockerfile时运行命令，父镜像在被子继承后父镜像的onbuild被触发。
