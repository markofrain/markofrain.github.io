Oracle PL/SQL 游标

2018-06-27
Oracle

###游标的概念 游标的使用可以让用户像操作数组一样操作查询出来的数据集。 把游标形象的看成一个变动的光标，实际上是一个指针，它在Oracle存放数据查询结果集的内存中，每当指针指向一条记录，就可以得到所指向的数据，初始时，它指向第一条记录的之前

游标可分为静态游标和REF游标

静态游标就像是一个数据快照，打开游标后的结果集是对数据数据的备份，数据不随着对标执行DML操作而改变。

静态游标

包括两种类型：显式游标和隐式游标 1. 显示游标 显式游标是用户或开发者管理的游标，在使用前必须有明确的游标声明和定义，这样的游标会关联数据查询语句，打开游标后，用户可利用游标的位置对结果集检索，返回单一行记录，关闭游标后不能进行任何操作。一切由用户自己完成。

  1. 隐式游标 隐式游标也成为SQL游标，由PL/SQL构造和管理，该游标用户无法控制，但能得到它的属性信息。

显示游标

通过显式游标，用户可以操作返回的数据。 通常以如下方式处理查询结果集:

  • 打开显式游标(OPEN语句)，从结果集中获取行(FETCH语句)，关闭显式游标(CLOSE语句)
  • 在FOR LOOP语句中使用显式游标，更加方便

  1. 要使用游标必须声明并定义游标变量

    —声明游标
    CURSOR cursor_name [parameter_list] RETURN return_type;
    —定义游标
    CURSOR cursor_name [ parameter_list ] [ RETURN return_type ]
    IS select_statement;

示例:游标声明

DECLARE
    CURSOR c1 RETURN product%ROWTYPE;       --声明c1

    CURSOR c2 IS SELECT *  FROM product WHERE productPrice>10;      --声明并定义c2,省略返回类型


BEGIN
    NULL;       --NULL表示跳过此处
END;

  1. 打开游标 对数据集操作，必须先打开游标初始化了游标指针，一旦打开，其结果集都是静态的

    OPEN cursor_name;

  1. 提取游标，读取数据 该语句检索结果集的当前行，将该行的列值存储到变量或记录中，并将游标指针移动到下一行。一般与循环语句一起使用，使用FETCH时游标属性%ROWCOUNT会不断累加。

    FETCH cursor_name INTO variables;

  1. 关闭游标 关闭后无法从结果集中提取记录或引用其属性，如果尝试操作则引发预定义异常INVALID_CURSOR。

当然你可以重新打开一个已经关闭的游标，在重新打开它之前，你必须关闭显示游标，否则引发CURSOR_ALREADY_OPEN预定义异常。

CLOSE cursor_name;

示例：使用LOOP语句提取游标 获得产品名称和产品价格

DECLARE
    v_name product.productName%TYPE;
    v_price product.productPrice%TYPE;
    CURSOR cur_nameAndPrice IS SELECT  productName,productPrice FROM product;
BEGIN
    OPEN cur_nameAndPrice;
    LOOP
        FETCH cur_nameAndPrice INTO v_name,v_price;
        EXIT WHEN cur_nameAndPrice%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE(v_name || '----' || v_price);
    END LOOP;
    CLOSE cur_nameAndPrice;
END;
显式游标的属性

利用游标属性可以获得游标执行的相关信息。

  • %ISOPEN：用于判断游标是否打开，如果已经打开则返回TRUE，否则返回FALSE。
  • %FOUND：检测行数据是否有效，有效则返回TRUE，否则返回FALSE
  • %NOTFOUND：与%FOUND相反。
  • %ROWCOUNT：累积到当前为止，使用FETCH提取数据的行数。

示例:显式游标属性

DECLARE
    v_product product%ROWTYPE;
    CURSOR cur_product IS SELECT *  FROM product;
BEGIN
    OPEN cur_product;                                       --打开游标
    IF cur_product%ISOPEN THEN                  --判断是否打开游标
        LOOP                                                        ---循环
            FETCH cur_product INTO v_product;       ---提取数据
            IF cur_product%FOUND THEN
                DBMS_OUTPUT.PUT_LINE('第' || cur_product%ROWCOUNT || '条数据,产品名称=' || v_product.productName || ',价格=' || v_product.productPrice);
            ELSE
                EXIT;                                       --当%FOUND返回FALSE时，说明无数据了，退出循环
            END IF;
        END LOOP;
    END IF;
    CLOSE cur_product;
END;
BULK COLLECT提取数据

通常使用FETCH…INTO…语句提取数据，这种方式是提取单条数据，在数据量大的情况下，执行效率不是很理想。而BULK COLLECT语句可以批量提取数据，在数据量大的情况下，效率会比单条数据高。 示例：BULK COLLECT提取数据

DECLARE
    TYPE proc_tab IS TABLE OF product%ROWTYPE;      --定义索引表
    v_product proc_tab;                                                     --声明索引表变量
    CURSOR cur_product IS SELECT *  FROM product;   --声明并定义游标变量
BEGIN
    OPEN cur_product;

    LOOP                                                                            --外层LOOP批量提取数据
        FETCH cur_product BULK COLLECT INTO v_product LIMIT 2;  --批量提取两条数据
        FOR i IN 1..v_product.count LOOP                                    --内层FOR LOOP对一次批量提取的数据进行循环处理
            DBMS_OUTPUT.PUT_LINE('产品ID=' || v_product(i).productId || '，产品名称=' || v_product(i).productName || ',产品价格=' || v_product(i).productPrice);
        END LOOP;
        EXIT WHEN cur_product%NOTFOUND;                     --无数据后退出
    END LOOP;

    CLOSE cur_product;
END;

该案例使用索引表来存储每次提取出的数据，LIMIT 表示每次提取2条数据，v_product.count表示获得索引表中的记录数。

使用CURSOR FOR LOOP循环简化游标的读取

使用 CURSOR FOR LOOP的方式，它并不需要特别声明变量。不需要手动打开游标，且自动从集中获取行，处理完成后自动关闭。

DECLARE
    CURSOR cur_product  IS SELECT *  FROM product;
BEGIN
    FOR prod_record IN cur_product LOOP
        DBMS_OUTPUT.PUT_LINE('产品ID=' || prod_record.productId || '，产品名称=' || prod_record.productName || ',产品价格=' || prod_record.productPrice);
    END LOOP;
END;

FOR LOOP循环中的prod_record变量就是自动声明的记录变量，类型为%ROWTYPE。 注意：该CURSOR FOR LOOP与循环结构中的FOR LOOP语句是不一样的，所以在循环控制中，关于游标的FOR LOOP没有说，才放到这里解释。

带参数的游标

使用显示游标可以指定参数，指定的参数包括参数的顺序和参数的类型。参数可以传递给游标在查询是使用，这样就方便了使用不同条件的查询，也方便了游标在存储过程中使用。 简而言之，如果你学过Java语言或其他语言，那么他就相当于方法差不多的情况，具有形参和实参的说法，定义游标时的形参以及FETCH游标时传递的实参。 示例：带参数的游标

DECLARE
    v_name product.productName%TYPE := '晨光';    --程序变量
    v_product product%ROWTYPE;                      ---行记录类型
    CURSOR cur_product (name VARCHAR2)      ---声明定义带参数的游标
        IS SELECT *  FROM product WHERE productName LIKE name||'%';
BEGIN
    OPEN cur_product(v_name);                   --添加实参并开启游标

    LOOP
        FETCH cur_product INTO v_product;   
        EXIT WHEN cur_product%NOTFOUND;
        DBMS_OUTPUT.PUT_LINE('产品ID=' || v_product.productId || '，产品名称=' || v_product.productName || ',产品价格=' || v_product.productPrice);
    END LOOP;

    CLOSE cur_product;
END;

当然可以添加默认值，如果不在开启游标时传递实参的话将使用默认值。 示例：默认参数

----游标声明定义处
CURSOR cur_product (name VARCHAR2 DEFAULT 'HUAWEI')         ---声明定义带参数的游标
        IS SELECT *  FROM product WHERE productName LIKE name||'%';

----开启游标处
OPEN cur_product;

当然，如果定义时有参数，且没有默认值，而开启游标时不赋予实参，则会发生错误。

隐式游标

每当运行SELECT或DML语句时，PL/SQL会打开一个隐式游标，隐式游标受用户控制，隐式游标与显式游标的不同之处:

  • 隐式游标由PL/SQL自动管理
  • 隐式游标的默认名称是SQL
  • SELECT或DML操作产生隐式游标
  • 隐式游标的属性值始终是最新执行的SQL语句

隐式游标的属性 虽然与显式游标属性相似，但却有不同之处。

  • SQL%ISOPEN属性：总返回FALSE，隐式游标在关联语句运行后总是关闭
  • SQL%FOUND属性：是否受到行影响。没有SELECT或DML语句运行则返回NULL，如果SELECT或DML返回影响多行数据则返回TRUE，否则返回FALSE
  • SQL%NOTFOUND：当没有受影响行时，返回TRUE，否则返回FALSE；
  • SQL%ROWCOUNT：返回受影响的行数

  1. 示例:SQL%FOUND

    DECLARE
    v_product product%ROWTYPE;
    BEGIN
    SELECT * INTO v_product FROM product WHERE productId=1;
    IF SQL%FOUND THEN

    DBMS_OUTPUT.PUT_LINE('得到受影响行数，受影响行数为' || SQL%ROWCOUNT);

    ELSE

    DBMS_OUTPUT.PUT_LINE('未收到行影响');

    END IF;
    END;

  1. SQL%NOTFOUND属性 SQL%NOTFOUND与SQL%FOUND相反。

但SQL%NOTFOUND属性对PL/SQL SELECT INTO语句无用。因为如果SELECT INTO不返回行会先引发NO_DATA_FOUND预定义异常，然后才能检查SQL%NOTFOUND。而如果只要使用聚合函数总会返回一个值，则SQL%NOTFOUND总是为FALSE。

  1. SQL%ROWCOUNT SQL%ROWCOUNT属性的值与事务的状态无关。因此：

  • 当事务回滚到保存点时，值SQL%ROWCOUNT不会恢复到保存点之前的值。

  • 当事务结束时，SQL%ROWCOUNT不会恢复到父事务中的原始值。

    CREATE TABLE product_temp AS SELECT * FROM PRODUCT;

    DECLARE

    v_num number(6) := 5;

    BEGIN

    DELETE FROM product_temp WHERE productId=v_num;
DBMS_OUTPUT.PUT_LINE('删除产品数量' || to_char(SQL%ROWCOUNT));
COMMIT;

    END;

PL/SQL异常

PL/SQL运行过程中可能出现错误，有程序本身原因或者来自法开发人员自定义的数据，而所有的这些错误我们称为异常。 为了使程序更具有健壮性，需要捕获这些异常并进行处理。当异常发生时，程序会调到异常块处，将控制权限交给异常处理程序。 例如一个数除以0时，会导致异常发生，并提示出错误号。而除以0的异常属于Oracle内部定义异常。 处理异常的语法:

EXCEPTION
    WHEN exception1 [OR exception2...] THEN
        statement...
    WHEN exception3 [OR exception4...] THEN
        statement...
    [WHEN OTHERS THEN statement..]

语法解释:

  • EXCEPTION表示异常块的关键字
  • WHEN exception THEN则是用来匹配异常名称，并执行对应的语句
  • WHEN OTHERS THEN则表示当其他异常都不匹配时，执行对应的语句

这个异常代码放在那，在最开始PL/SQL结构中就写了,加上EXCEPTION,通常就是这样的:

DECLARE
BEGIN
EXECEPTION
END;

Oracle中的异常分为三类:

  • 预定义异常
  • 非预定义异常(官网的说法是内部定义异常)
  • 自定义异常

预定义异常

Oracle中为每个错误提供一个错误号，而捕获异常则需要异常有名称。Oracle提供了一些已经定义好名称的常用异常，就是预定义异常。

当然，预定义异常也不止这些，这些只是在开发中比较常用的异常。到Oracle12c为止，一共提供了25中预定义异常。利用下面的语句可查看Oracle的预定义异常。

SELECT *  FROM DBA_SOURCE WHERE NAME='STANDARD' AND TEXT LIKE '%EXCEPTION_INIT%';

示例: 就拿除以0的例子.

DECLARE
    v_num number(8);
BEGIN
    v_num := 20/0;
EXCEPTION
    WHEN ZERO_DIVIDE THEN
        DBMS_OUTPUT.PUT_LINE('除数为0');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('其他异常');
END;

当除数出现0时，程序会马上进入异常捕获部分，当发生的异常和列表中的异常名称匹配成功时，执行对应语句，异常匹配的顺序是从上往下的,所以尽量把合适的异常放在第一位。

非预定义异常(内部定义异常)

Oracle中有很多异常都是非预定义异常。他们只有错误编号和错误描述，而没有名称是不能被异常捕获的。 为了解决该问题，Oralce允许开发人员为这些异常添加一个名称，使它们被异常捕获到。 注：用户声明名称的内部定义异常仍然是内部定义的异常，而不是用户定义的异常 为一个非预定义异常定义名称需要如下步骤: 1. 声明一个异常的名称 2. 把名称与异常编号相互关联 声明部分定义异常名称:

exception_name EXCEPTION;

将定义好的异常与错误编号关联起来，需使用如下语句

PRAGMA EXCEPTION_INIT(exception_name,exception_code)；

示例:

DECLARE
    v_id provider.providerId%TYPE :=456;
    my_exception EXCEPTION;
    PRAGMA EXCEPTION_INIT(my_exception, -2291);
BEGIN
    UPDATE PRODUCT SET providerId = v_id where productId = 8;
EXCEPTION
    WHEN my_exception THEN
      DBMS_OUTPUT.PUT_LINE('违反完整约束条件');
      DBMS_OUTPUT.PUT_LINE(SQLCODE);
      DBMS_OUTPUT.PUT_LINE(SQLERRM);
      ROLLBACK;
END;

这样可以捕获异常，可能你会不太清楚SQL CODE是多少，没办法关联。我似乎也没有找到办法。只能在开发时尝试通过SQLCODE函数获得错误号，SQLERRM获得错误信息。

自定义异常

通常我们想自己定义异常，来控制自己的业务情况，比如产品数量不为负数，生产日期等，预定义和非预定义都无法捕获。 语法:

--声明异常
exception_name EXCEPTION;
--抛出异常
RAISE exception_name;

自定义异常示例:

DECLARE
    v_price product.productPrice%TYPE;      ---产品数量
    my_price_excep EXCEPTION;
BEGIN
    select productPrice INTO v_price FROM product WHERE productId=1;
    IF v_price<=0 THEN
        RAISE my_price_excep;
    END IF;
EXCEPTION
    WHEN my_price_excep THEN
        DBMS_OUTPUT.PUT_LINE('价格不正确，需大于0');
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('出现其他错误');
END;

该例子看起来似乎没有什么意义，只是介绍如何使用。

重新定义错误消息

RAISE_EXCEPTION_ERROR存储过程可以重新定义异常错误信息。用于抛出一个异常并给异常赋予一个错误号以及错误信息。

RAISE_EXCEPTION_ERROR(error_number,error_message[,{ TRUE | FALSE }]);

错误号的范围是-20,000到-20,999。错误信息是文本字符串，最多为2048字节。 TRUE和FALSE表示是添加(TRUE)进错误堆(ERROR STACK)还是覆盖(overwrite)错误堆(FALSE)。缺省情况下是FALSE。 示例:

DECLARE
    v_num number(8);
BEGIN
    v_num := 20;
  IF v_num>10 THEN
    raise_application_error(-20000,'错误');
  END IF;
END;

PL/SQL中使用DML和DDL语言

PL/SQL中除了可以执行查询语句外，还可以执行DML数据操纵语言和DDL数据定义语言。

DML数据操作语言语句的使用

示例: 添加一条新纪录，编号为表中最大编号+1

DECLARE
    v_new_id product.productId%TYPE;
BEGIN
    SELECT MAX(productId) INTO v_new_id FROM product;
    v_new_id := v_new_id+1;
    INSERT INTO product VALUES(v_new_id,'MacBook Pro 13.3',12000,1500,NULL);
    COMMIT;
END;

DDL数据定义语言的使用

要想在PL/SQL中使用，需要一条命令来执行，该命令是EXECUTE IMMEDIATE,利用它可以执行动态的SQL语句，也就是不仅可以执行DDL语句，也可以执行DML语句。 示例

DECLARE
    prod_createStr varchar2(200);
BEGIN
    prod_createStr := 'CREATE TABLE TAB_STUDENT
                                (
                                    STUDENTID number(8) primary key,
                                    STUDENTNAME varchar2(20),
                                    AGE nchar(2)
                                )';

    EXECUTE IMMEDIATE prod_createStr;
END;


DECLARE
  v_alter_address varchar2(200);
BEGIN
  v_alter_address := 'ALTER TABLE TAB_STUDENT ADD ADDRESS VARCHAR2(50)';

  EXECUTE IMMEDIATE v_alter_address;
END;

需要注意，将数据定义语言作为字符串，通过EXECUTE IMMEDATE动态执行的时候，最后的分号要去掉，否则错误，这是与一般执行DDL的不同之处。 通常情况下，利用EXECUTE IMMEDATE命令执行DDL语句用在存储过程当中。

PL/SQL结构控制

IF条件控制

条件语句，控制流程 IF语句的三种方式:IF…、IF…ELSE…、IF…ELSIF…。根据实际业务灵活选择。 语法:

IF condition THEN
    statements;
END IF;


IF condition THEN
    statements;
 ELSE
    statements;
 END IF;


IF condition THEN
    statements;
ELSIF condition THEN
    statements;
...
[ELSE statements;]
END IF;

  1. 对于这三种IF的使用不需要多说，不管什么面向对象的解释语言，基础都有控制结构的。

  2. 而你只需要记住语法结构，最后会有END IF。而ELSIF一定不要写错。

  3. 关于数值表达式+、-、、/、*都可以用。以及关系表达式和逻辑表达式NOT、OR、AND。

  4. condition可以使用任何表达式和变量，常量等。

CASE条件控制

与IF语句有同样的效果。

简单CASE控制

只有当CASE子句中表达式的值等于某个WHEN的表达式的值时，才执行响应的THEN子句。 语法:

[<<label_name>>]
CASE case_operand
WHEN when_operand THEN
    statements;
[
    WHEN when_operand THEN
        statements;
] ...
[ELSE statements]
END CASE [label_name];

语法说明:

  • <>：一个CASE标签，可用于结束当前这个CASE语句。
  • case_operand：表达式，通常是一个变量。除了BLOB、BFILE、对象类型、记录类型、索引表、变长数组或嵌套表，其他类型都允许。
  • when_operand ：case_operand对应结果，如果两者值相同，则执行statement语句。可以有多个WHEN..THEN
  • ELSE statement：当所有值都不能对应上时，执行此处的语句。

CASE示例 模拟获得当前日期得到月份

DECLARE
  v_month nvarchar2(3);
BEGIN
  SELECT trim(to_char(sysdate,'MONTH')) INTO v_month FROM product WHERE productId=1;

  CASE v_month
  WHEN '1月' THEN
    DBMS_OUTPUT.PUT_LINE('一月');
  WHEN '2月' THEN
    DBMS_OUTPUT.PUT_LINE('二月');
  WHEN '3月' THEN
    DBMS_OUTPUT.PUT_LINE('三月');
  WHEN '4月' THEN
    DBMS_OUTPUT.PUT_LINE('四月');
  WHEN '5月' THEN
    DBMS_OUTPUT.PUT_LINE('五月');
  WHEN '6月' THEN
    DBMS_OUTPUT.PUT_LINE('六月');
  ELSE
    DBMS_OUTPUT.PUT_LINE('不在一到六月');
END CASE;

END;

CASE语句执行的顺序是从上往下的顺序，当WHEN子句的语句被执行后，CASE结构就执行完毕。

搜索式的CASE语句

这种与IF非常非常类似,将WHEN子句的表达式只能是布尔值，布尔表达式。这与IF相同 语法:

[<<label_name>>]
CASE
WHEN boolean_expression THEN statement;
[WHEN boolean_expression THEN statement;]...
[ELSE statement];
END CASE [label_name];

LOOP循环控制

LOOP语句也就是循环语句，能重复的执行指定的语句。有四种形式的LOOP。

  • LOOP;
  • WHILE…LOOP;
  • FOR…LOOP;
  • CURSOR FOR LOOP;
基本LOOP循环

基本的LOOP语句本身没有包含中断循环的条件，通常情况下利用EXIT、GOTO等可以中断LOOP循环使用。异常也使LOOP语句中断。 基本LOOP循环语法:

[<<label_name>>]
LOOP
    statement...
END LOOP;
1. 通过IF语句和EXIT的组合结束循环

EXIT必须在循环体内部，它可以是循环正常无条件终止循环，而终止循环后会正常执行循环外的语句 操作示例: 循环变量，当变量值大于某值时退出循环。

DECLARE
    v_num NUMBER(8) := 1;       --声明并初始化变量
BEGIN
    <<basic_loop>>
    LOOP
        v_num := v_num+1;   --循环中，自增1
        IF v_num>5 THEN         --判断条件，是否需要退出
            EXIT basic_loop;
        END IF;
    END LOOP;
    DBMS_OUTPUT.PUT_LINE('LOOP循环已经结束,v_num的值为' || v_num);
END;

一般像有多个嵌套的循环的话，建议还是加上LOOP标签的，这样能更直观的看出退出的是哪个循环

2. 通过EXIT…WHEN语句退出循环

就是和IF EXIT差不多意思的退出，当什么条件的时候退出。没有loop标签也是可以的。

DECLARE
    v_num NUMBER(8) := 1;       --声明并初始化变量
BEGIN
    <<basic_loop>>
    LOOP
        v_num := v_num+1;   --循环中，自增1
        EXIT basic_loop WHEN v_num>5;       
    END LOOP;
    DBMS_OUTPUT.PUT_LINE('LOOP循环已经结束,v_num的值为' || v_num);
END;
WHILE…LOOP循环语句

该语句本身可以终止LOOP循环，当WHILE后的布尔值为TRUE时，将执行一次，然后重新判断是否为TRUE，否则直接退出循环。 语法结构:

[<<label_name>>]
WHILE boolean_expression
LOOP
    statement...
END LOOP [label_name];

WHILE…LOOP示例 输出20以内被3整除的数

DECLARE
    v_num NUMBER(8) := 1;
BEGIN
    WHILE v_num<=20
    LOOP
        IF MOD(v_num,3) = 0 THEN
            DBMS_OUTPUT.PUT_LINE(v_num);
        END IF;
        v_num := v_num+1;
    END LOOP;
END;
FOR…LOOP循环语句

语句循环遍历指定范围内的整数，主要用于固定次数的循环。 语法:

[<<label_name>>]
FOR index_name IN [REVERSE] lower .. upper LOOP
    statement..
END LOOP [label_name];

语法说明

  • index_name：循环计数器，通过该值得到当前的循环次数。
  • REVERSE：指定循环方式，从lower到upper，还是相反。使用REVERSE关键字，循环方式正好相反
  • lower..upper：循环范围的下标界到上标界，中间的两个点是必须的，不能省略。

FOR…LOOP示例: 1-20的整数的和

DECLARE
    v_num number(8) := 0;
BEGIN
    FOR idx IN 1..20 LOOP
        v_num := v_num + idx;
    END LOOP;
    DBMS_OUTPUT.PUT_LINE('LOOP循环结束，值为' || v_num);
END;

对于循环范围lower和upper可以使用数值型的变量代替，而当两者相等时，只循环一次。

Oracle PL/SQL介绍，声明部分的标量和复合变量

2018-06-26
Oracle

PL/SQL

什么是PL/SQL

PL/SQL(Procedural Language/Structured Query Language)是Oracle公司在标准SQL语言基础上进行拓展而形成的一种可以在数据库上进行设计编程的语言，通过Oracle公司的PL/SQL引擎执行。 PL/SQL完全可以像Java语言一样实现逻辑判断、条件循环以及异常处理等。 PL/SQL有以下几个特点:

  • 支持事务控制和SQL数据操作命令
  • 支持SQL的数据类型、并且在此基础上扩展了新的数据类型，也支持SQL的函数以及运算符。
  • PL/SQL可以存储在Oracle服务器中
  • Oracle有自己的DBMS包，可以处理数据的控制和定义命令

PL/SQL的优势

  1. 提高程序运行性能 标准的SQL被执行时，只能一条一条的向服务器发送,多次连接数据库服务器很耗费资源。如果使用PL/SQL，PL/SQL的语句块可以包含多条SQL语句，而语句块可以嵌入到程序中，甚至存储在Oracle服务器上，用户只需一次连接，就可以在Oracle内部执行。

  2. 使程序模块化 在程序块中可以实现一个或几个功能。我们可以把多张表的操作都放在一个块里，而对外只需要提供一个调用方法和需要传递的参数。这种操作就是通过存储过程完成。

  3. 采用逻辑控制语言控制程序结构 PL/SQL语句块中可以通过if判断，循环等等，通过限制语句的执行，从而完成复杂的业务。

  4. 利用处理运行时的错误信息 标准SQL在遇到错误时会提示异常，但提示却非常不友好，并不能直接告诉你错误的准确位置，并且批量执行SQL出现错误时导致SQL执行终止。而PL/SQL语句块中可以处理一些程序上的异常，不至于因SQL操作，而造成调用的SQL页出现错误提示。

PL/SQL的结构

PL/SQL程序的基本单位是块(block)，PL/SQL块很明确的分三部分，其中包括声明部分、执行部分和异常处理部分。

[DECLARE]       --声明部分的关键字
                        /*声明部分的代码，用于声明变量、常量以及类型*/
BEGIN               --执行部分的关键字
                        /*执行部分，也是PL/SQL的主体部分，包括SQL语句以及逻辑控制语句*/
[EXCEPTION] --异常部分关键字
                        /*异常处理部分，当出现异常时，程序将进入此处*/
END;                    --执行结束关键字，程序结束

示例:

BEGIN
    DBMS_OUTPUT.PUT_LINE('这是执行部分');
END;


DECLARE
    v_result NUMBER(8,2);
BEGIN
    v_result := 100/6;
    DBMS_OUTPUT.PUT_LINE('结果是:' || v_result);
END;

PL/SQL基本规则

  • 标识符不分大小写
  • 标识符中只允许字母、数字、下划线、字母开头
  • 不能使用保留字
  • 语句使用分号结束

建议要求:

  • 每行只写一条语句
  • 全部保留字、关键字、内置函数、程序包以及用户定义的数据类型都是用大写
  • 所有过程名大写
  • 命名以”_”连接，且均有意义，看下图命名方式
  • 建议添加注释

注释 1. 单行注释 使用两个短横线开头”–”,可以注释掉后面的语句

  1. 多行注释 使用“/…/”可以注释掉中间的部分，与java多行注释一样。

示例

DECLARE
    v_result NUMBER(8,2);
BEGIN
    --为变量赋值
    v_result := 100/6;
    /*
        打印结果
    */
    DBMS_OUTPUT.PUT_LINE('结果是:' || v_result);
END;

PL/SQL声明部分，标量，复合变量的使用

声明部分变量，常量使用

  1. 变量声明语法结构

    variable_name datatype
    [

    [NOT NULL]
{:= | DEFAULT} expression;

    ];

语法说明:

  • variable_name:变量名称
  • datatype:变量的数据类型
  • NOT NULL:非空约束
  • {:= | DEFAULT} express : “:=”表示为变量在声明时直接赋值的符号，而DEFAULT表示无值,默认值的关键字。值为express,可以是表达式或函数

  1. 常量声明语法结构

    constant_name CONSTANT datatype
    [NOT NULL]
    { := | DEFAULT} expression;

结构与常量类似，只不过多了一个常量关键字CONSTANT。

标量类型的变量

表量类型的变量是最简单类型的变量，也是最常用的变量类型，它本身是一个单一值，不包含任何的类型组合。标量类型主要包含数值类型、字符类型、布尔类型和日期类型，还有一种特殊声明变量类型的方式，就是利用%TYPE。 其他的不再说，就说一下 %TYPE方式定义变量类型。 %TYPE引用: 它利用已存在的数据类型来定义新数据的数据类型。例如：当定义多个相同类型的变量常量时，第一个定义变量，后面的只需要利用%TYPE引用。最常见的就是表中的字段类型作为变量或常量的数据类型,这样可以保证定义变量类型可以与表中字段类型同步，避免字段类型发生变化而修改。 示例:

DECLARE
  v_productId product.productId%TYPE;       --产品编号
  v_productName varchar2(18);                   --产品名称
  v_productPrice number(8,2);                       --产品价格
  v_test CONSTANT v_productName%TYPE:='测试';
  v_date date DEFAULT sysdate;                  --默认值为当前日期
BEGIN
  SELECT productId,productName,productPrice INTO v_productId,v_productName,v_productPrice FROM product WHERE productId=1;

  DBMS_OUTPUT.PUT_LINE('v_productId=' || v_productId);
  DBMS_OUTPUT.PUT_LINE('v_productName=' || v_productName);
  DBMS_OUTPUT.PUT_LINE('v_productPrice=' || v_productPrice);
  DBMS_OUTPUT.PUT_LINE('v_test=' || v_test);
  DBMS_OUTPUT.PUT_LINE('v_date=' || v_date);
END;

代码说明: 1. 代码运行在PL/SQL Developer软件中，新建SQL窗口，SQL页执行，输出页查看输出内容。 2. 代码第二行表示v_productId的类型为product表中中productId字段的类型 3. 代码第五行表示声明常量v_test，变量类型为v_productName的变量类型 4. 代码第六行表示变量v_date为date类型，且默认值为sysdate函数的值 5. 变量不仅可以在声明部分赋值也可以在主体BEGIN中赋值，但是前提是必须声明了才能赋值。 6. BEGIN中SELECT代码，INTO关键字，表示将前面的表字段值赋值给后面的变量，顺序需要一一对应，且只能返回一行数据 7. 最后几行DBMS库的PUT_LINE来输出一行，在输出选项框查看。

复合类型的变量

复合类型就是，每个变量包含几个元素，可以存储多个值。与标量类型有差异，需先定义才能声明。 最常用的三种类型，记录类型，索引表类型，VARRAY数组。

记录类型

该类型可以包含多个成员，每个成员的类型可以不同，成员可以是标量类型，也可以是引用其他变量的类型(%TYPE)。 该类型比较适合处理查询语句中有多个列的情况，就如上面的例子，多个列都在同一张表或多张表中。 然后想要得到成员的值，可以通过“变量名称.成员名称”的方式。 两种声明方式: 1. 记录定义 语法结构:

TYPE type_name IS RECORD
(
field_name datatype
[ [NOT NULL] { := | DEFAULT } expression ]

[,...]
);

语法说明:

  • TYPE关键字表示类型，IS是固定的关键字，RECORD关键字证明了定义该类型为记录类型。
  • 成员属性的写法和变量声明是一样的，可以有多个成员属性
  • 另外，注意，跟创建表差不太远，成员属性后跟的是逗号，不是分号了。

示例:定义“记录类型”变量

DECLARE
TYPE product_record IS RECORD
(
  v_productId product.productId%TYPE,       --编号
  v_productName varchar2(18),               --名称
  v_productPrice number(8,2)                --价格
);
v_product product_record;
BEGIN
    SELECT productId,productName,productPrice INTO v_product FROM product WHERE productId=1;

    DBMS_OUTPUT.PUT_LINE('v_productId=' || v_product.v_productId);
    DBMS_OUTPUT.PUT_LINE('v_productName=' || v_product.v_productName);
    DBMS_OUTPUT.PUT_LINE('v_productPrice=' || v_product.v_productPrice);
END;

代码解释:注意在定义时，不要忘记成员属性后加逗号，定义结束后加分号，少一个符号都会导致错误。尽管已经定义了记录类型，但是仍然摆脱不了顺序的问题，仍需要跟定义记录中的成员属性顺序保持一致。

  1. 使用%ROWTYPE声明记录类型数据 利用%ROWTYPE，这种声明方式可以直接引用表中的行作为变量类型，与%TYPE类似，它将得到整个行的数据类型。

%ROWTYPE示例

DECLARE
    v_product product%ROWTYPE;
BEGIN
    SELECT *  INTO v_product FROM product WHERE productId=2;

    DBMS_OUTPUT.PUT_LINE('v_productId=' || v_product.productId);
    DBMS_OUTPUT.PUT_LINE('v_productName=' || v_product.productName);
    DBMS_OUTPUT.PUT_LINE('v_productPrice=' || v_product.productPrice);
END;

代码说明:变量v_product的数据类型是表product的行记录类型，用%ROWTYPE方式声明了变量数据类型。仅此可以通过*把所有字段都赋值给v_product,而调用时，因为使用的是product的行记录类型，所以直接“.表字段名”即可。 这张方法较为方便，不过如果表字段很多很多，只需要用到几个字段的话，就未免有些浪费了。

索引表类型

该类型与数组类似，利用键值查找对应的值。但这里的键可以使用字符串，且长度不固定而自动增长。 键可以是数值或字符串，值可以是标量类型或记录类型。如果某键已有对应数据，那么将会覆盖更改。 定义语法:

TYPE type_name IS TABLE OF
{
    column_type | variable_name%TYPE | table_name.colum_name%TYPE | table_name%ROWTYPE
}
[NOT NULL]
INDEX BY {PLS_INTEGER | BINARY_INTEGER | VARCHAR2(v_size)};

语法解释： 1. 花括号中表示索引表中的数据类型，可以是标量类型，通过%TYPE引用变量的类型，表字段的类型和%ROWTYPE行记录类型。 2. INDEX BY后面表示数组下标的数据类型。PLS_INTEGER和BINARY_INTEGER是属于数值类型，也算是标量类型中。 3. 这只是定义，接下来就可以声明变量了。 PLS_INTEGER和BINARY_INTEGER类型延伸

两个类型通常可以认为是一样的类型。存储有符号的整形值，范围在，-2^31~2^31-1之间。 不一样的地方就是BINARY_INTEGER发生溢出时能指派NUMBER类型(不超出NUMBER精度范围)而不至于发生异常,而PLS_INTEGER运算溢出时会触发异常，建议使用PLS_INTEGER。 SIMPLE_INTEGER类型属于PLS_INTEGER的子类型取值范围与PLS_INTEGER相同，只是该类型不允许为空。如果数据本身不需要溢出检查而且也不可能为空，则选择该类型，该类型性能比PLS_INTEGER高。

示例:

DECLARE
TYPE product_table_fst IS TABLE OF product%ROWTYPE      --定义索引表，类型%ROWTYPE,索引类型PLS_INTEGER
    INDEX BY PLS_INTEGER;
TYPE product_table_sec IS TABLE OF nvarchar2(20)                --定义索引表，类型nvarchar2(20),索引类型PLS_INTEGER
    INDEX BY PLS_INTEGER;

v_product_row product_table_fst;
v_char product_table_sec;
BEGIN
    v_char(1) := '第一位';
    v_char(-1) := '也可以是正数或负数，但不能溢出';

    SELECT *  INTO v_product_row(1) FROM product WHERE productId=1;

    DBMS_OUTPUT.PUT_LINE('行数据v_product_row(1)的值为' || '---' || v_product_row(1).productId || v_product_row(1).productName);
    DBMS_OUTPUT.PUT_LINE('v_char(1)---' || v_char(1));
    DBMS_OUTPUT.PUT_LINE('v_char(-1)---' || v_char(-1));
END;
VARRAY数组

该类型的元素个数时需要限制的，是一个存储有序元素的集合。集合下标从1开始，适合较少数据的使用。 语法声明:

TYPE type_name IS   { VARRAY | VARYING ARRAY} (size)
    OF element_type [NOT NULL];

语法说明:

  • type_name : VARRAY数组名称
  • VARRAY | VARYING ARRAY：两者一个意思，varying array简写为varray
  • size：数组长度
  • element_type：数组中元素类型

示例:

DECLARE
    TYPE varr IS VARRAY(100) OF product.productName%TYPE;

    v_test varr := varr('a','b');
BEGIN
    SELECT productName INTO v_test(1) FROM product WHERE productId=3;
    v_test(2) := 'This chanage';

    DBMS_OUTPUT.PUT_LINE('productName--' || v_test(1));
    DBMS_OUTPUT.PUT_LINE('v_test(2)--' || v_test(2));

END;

第三行v_test赋初始值，varr是VARRAY数组的类型名称，通过上面那种方式，直接赋值。使用 变量名(i)赋值或调用。
