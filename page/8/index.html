路由

Navigator是一个路由管理的组件，它提供了打开和退出路由页方法。Navigator通过一个栈来管理活动路由集合。通常当前屏幕显示的页面就是栈顶的路由。Navigator提供了一系列方法来管理路由栈。

Navigator提供了一系列方法来管理路由栈，在此我们只介绍其最常用的两个方法：

  • Future push(BuildContext context, Route route)

    将给定的路由入栈(即打开新的页面)，返回一个Future对象，用以接收新路由出栈(即关闭)时返回的数据

  • bool pop(BuildContext context, [ result ])

    将栈顶路由出栈，result为页面关闭时返回给上一个页面的数据

Navigator 还有很多其它方法，如Navigator.replace、Navigator.popUntil等。

命名路由

路由配置，在MaterialApp添加属性routes，通过路由名和Widget对象，routes是一个map类型

在使用路由的时候通过Navigator.pushNamed(context, "new_page");push到新的路由界面

命名路由传参

Navigator.pushNamed的第三个参数是一个object类型的值，可传递参数

路由生成钩子

MaterialApp有一个onGenerateRoute属性，它在打开命名路由时可能会被调用。

当调用Navigator.pushNamed(...)打开命名路由时，如果指定的路由名在路由表中已注册，则会调用路由表中的builder函数来生成路由组件；如果路由表中没有注册，才会调用onGenerateRoute来生成路由。

axios携带请求头下载文件

2021-03-03
Vue2.0 / Javascript

以上代码request函数是封装的，内部还是使用axios设置参数，重要的属性是responseType设置为blob类型。

在调用时，通过URL.createObjectURL(blob);的方式获取blob转href的一个功能进行a标签的下载。

首先需要注意的是，跨域问题首先要解决，这不是主要的，因为其它接口也会出现跨域。

其次res.data.type和content-disposition的响应头值需要得到。这两个是响应头需要在后台设置。

第一行设置Content-Disposition包含文件名称

第二行设置ContentType，值一定是MediaType下带VALUE的字符串返回值，不要拿APPLICATION_OCTET_STREAM来获取。

第三行设置用于导出的响应头，因为需要拿到文件名,Content-Disposition属性是一定要前段拿到的。

在默认请求下，浏览器只能访问到默认的响应头

  • Cache-Control
  • Content-Language
  • Content-Type
  • Expires
  • Last-Modified
  • Pragma

其他属性需要通过Access-Control-Expose-Headers响应头设置要额外返回的响应头名字。

携带head时：

a标签download下载，非同源跨域，则download属性不起作用，若下载类型为浏览器不可直接预览的则依旧会自动下载。

get进行下载时，全局设置跨域后依然跨域(有时候好的有时候不行)。

post进行下载时，不跨域，但是无法弹出下载器进行下载(此问题不管是原生ajax还是封装的，只要res.data获取blob数据都会一次性将文件下载到浏览器内存，下载完成后拷贝到磁盘上，拷贝这个速度很快，如果下载的文件较大则会导致浏览器内存溢出并崩溃)。

携带认证头的可选择使用param参数的方式，加密到后台，并进行解密，使用两种方式混合。然后直接使用get方式进行加载就正常。

以上方式只适用于小文件。
