多master多slave同步集群

首先将rocketmq的bin配置到环境变量中，方便使用命令。

集群规划

编号 IP 角色 架构模式
1 192.168.38.130 nameserver,broker master1,slave2
2 192.168.38.131 nameserver,broker master2,slave1

在进行nameserver集群是无需配置文件，一般一台机器一个nameserver，直接启动即可。

配置多主多从相关文件。在conf下有2m-2s-sync、2m-2s-async、2m-noslave目录，分别对应多种多从同步模式，多主多从异步模式，多主无从模式

多主多从同步与异步配置基本一致，本配置以同步为例。

在2m-2s-sync目录下包含broker-a.properties broker-a-s.properties broker-b.properties broker-b-s.properties这些文件。不含-s的文件为master，否则为slave，但这只是一个名字的易于区分，具体角色在配置中的brokerRole属性进行区分。

以130的这两个配置为例，131的配置属性相同，需要在部署时需要注意的属性如下:

  • brokerClusterName：集群名字，多个broker配置相同的brokerClusterName组成一个broker集群
  • brokerName：broker名字可用来指定主从关系，如果同属于一个broker则brokerName应该相同，如何区分master和slave使用brokerId属性。
  • brokerId：同brokerName下的brokerId标识，如果为0则在当前brokerName中，该broker为master，大于0则为slave
  • brokerIP1：broker监听的IP，在多网卡ip的情况下需要显示设定ip，否则可能出现客户端连接失败的情况。
  • brokerIP2：主从关系存在时，slave会根据master的brokerIP2进行同步数据。
  • namesrvAddr：连接的nameserver地址，集群nameserver使用分号分开。
  • listenPort：broker监听端口，每个broker的端口在同一台机器上不能相同。
  • store*：数据存储目录，同一台机器上的broker的路径均不能重复。
  • brokerRole：当前broker配置文件定义了自身的角色。

brokerName相同可以认为是一组broker，但仍然是由master和slave组成的多个broker，仅仅是确定了主从关系。

集群分号问题

集群配置，在properties配置文件还是在设置namesrvAddr都是分号分隔不同地址。

存储消息路径问题

同一台机器下，如果有多个broker，则不同broker的存储路径不能一致，如某机器下有1个master和1个slave，则在配置broker-a.properties文件和broker-b-s.properties中的一下路径是不能冲突的。否则可能造成broker无法启动成功。

Exception in thread “main” org.apache.rocketmq.remoting.exception.RemotingTooMuchRequestException: sendDefaultImpl call timeout

集群启动完成并且console上可以看到集群，但是生产者无法连接到broker发送消息。

IP问题，如果服务器有多个ip，需要在properties或conf中设置brokerIP1=192.168.38.130指定ip即可,并且namesrvAddr也尽量设置准确地址。

brokerIP1 当前broker监听的IP
brokerIP2 存在broker主从时，在broker主节点上配置了brokerIP2的话,broker从节点会连接主节点配置的brokerIP2来同步。

默认不配置brokerIP1和brokerIP2时，都会根据当前网卡选择一个IP使用，当你的机器有多块网卡时，很有可能会有问题s

org.apache.rocketmq.client.exception.MQClientException: Not found the producer group connection For more information, please visit the url, http://rocketmq.apache.org/docs/faq/

在console上查询生产者页面信息时报错。原因为生产者生产完消息后不执行producer.shutdown();才行，否则无法查询到。

统称为主题。对于activemq的队列和消息区分有别。

activemq在最开始就定义当前消息组是队列还是主题。

rocketmq只向类型为top主题类型中发送消息，是队列还是主题由消费者来确定，负载均衡CLUSTERING则为队列模式，广播BROADCASTING则为主题模式。
