Maven中repository和mirror的使用

2021-02-24

原文参考自: https://blog.csdn.net/sky_tree_delivery/article/details/105415048

本文也是在使用到geotool工具时找到原文博客描述。在此记录一下非中央仓库引入jar包的问题。

repository称为仓库，是maven中储存jar包的站点，会提供pom文件、jar包等。为了使用户不配置任何仓库也可以加载资源，maven隐含地配置了maven的中央仓库（name属性为central），不论用户是否配置了自己的仓库。

repository仓库在项目pom中引入后可直接进行使用仓库下的jar依赖

但如果settings.xml文件中配置了,mirror标签，则mirror会对仓库进行拦截并代理下载。

mirror称为镜像，起拦截器的作用，本身不存储资源，而是指向一个资源地址（url）。镜像对项目中配置的仓库进行拦截，当一个镜像的mirrodOf参数匹配上了仓库的name参数，则进行资源映射，否则使用仓库中指定的url地址请求资源。

如果mirrorOf设置了*，则所有的jar依赖 都匹配到从此url中获取jar依赖，但aliyun中只提供中央仓库central的jar，因此部分通过其他仓库引入的jar就回获取失败。

所以可以认定为mirror的优先级别大于repository，不在mirrorOf中使用*匹配。
